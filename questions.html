Resposta:

A função verifica se o parâmetro fornecido é um vetor e se todos os elementos do vetor são numéricos com isNaN, se não, a função retorna uma mensagem indicando que o vetor não é composto somente por números. A função "Primo" verifica se um número é primo, utilizando um loop para testar se o número é divisível por algum número entre 2 e a raiz quadrada do número. Se o número for divisível por algum desses valores, significa que não é primo e retorna false, se não for divisível ai retorna true. Usando o método filter() do vetor arr junto com a função Primo para filtrar apenas os valores primos. O resultado é armazenado na constante primos. Caso não tenha números primos no vetor, a função retorna Null.

function filterPrimes(arr) {
if (!Array.isArray(arr) || arr.some(isNaN)) {
return "O vetor fornecido não é composto somente por números.";
}
function Primo(num) {
if (num <= 1) {
return false;
}
for (let i = 2; i <= Math.sqrt(num); i++) {
if (num % i === 0) {
return false;
}
}
return true;
}
const primos = arr.filter(Primo);
return primos.length > 0 ? primos : null;
}

A função deve procurar em cada objeto armazenado no no vetor, uma key (chave) que contenha um valor (value), fornecido. Nos Objetos Literais armazenados no vetor, deve existir as chaves "nome", "idade" e "data_nascimento". A primeira chave deve receber valores do tipo String; a idade, Number; e a última, Date. Caso a key fornecida como parâmetro não esteja presente nos objetos fornecidos, a função deve retornar o tipo especial "undefined". Caso o valor fornecido no parâmetro value não esteja presente nos objetos, deve ser retornando "null". Se houver objetos com o valor procurado, deve-se retornar um vetor que contenha esses objetos.

Resposta:

Uso de Array.isArray() para verificar se é um vetor. Se não for um vetor, exibe uma mensagem no console. Em seguida, verifica se a chave fornecida em key corresponde a uma das chaves esperadas: "nome", "idade" ou "data_nascimento". Se não corresponder, retorna undefined. Utilizando o método filter() para obter apenas os objetos cujo valor da chave seja igual ao valor fornecido em value, Então retorna resultado com o que foi encontrado ou null se nada foi encontrado.

function searchItems(arr, key, value) {
if (!Array.isArray(arr)) {
console.log("Parâmetro Inválido!");
return;
}
if (key !== "nome" && key !== "idade" && key !== "data_nascimento") {
return undefined;
}
const resultado = arr.filter(obj => obj[key] === value);
if (resultado.length === 0) {
return null;
}
return resultado;
}

{ nome: "Levi", idade: 42, data_nascimento: new Date(1981, 6, 1) },
{ nome: "Criso", idade: 50, data_nascimento: new Date(1973, 4, 2) },
{ nome: "Guilherme", idade: 45, data_nascimento: new Date(1978, 7, 20) }
];

const key = "idade";
const value = 42;

const resultado = searchItems(items, key, value);
console.log(resultado);

Questão 14

Mostre, exemplificando, qual a diferença entre Function Declaration, Function Expression e Arrow Function.

Resposta:

1 - Function Declaration é a forma padrão de declarar uma função de escopo global ou de função, sua sintaxe é simplesmente nomear a função e fornecer um bloco de código.

function soma(a, b){
return a + b;
}
console.log(soma(1, 1))

O resultado é (2)

2 - Function Expression é a definição da função dentro de um expressão, que é atribuido a uma variável, sendo utilizada também como argumento para outra função.
const subtra = function (a, b){
return a - b;
};
console.log(subtra(5, 5));

O resultado é (0)

3 - Arrow Function são usada para definir funções. São espressões de função que usam a sintaxe (=>), mas não têm valor próprio, elas herdam do contexto.
const multi = (a, b) => a*b; console.log(multi(2, 4));

O resultado é (8)

A diferença dessas três formas está em seu valor e na forma de declaração. A Function Declaration é mais versátil e pode ser chamada antes de sua definição, já as outras devem ser declaradas antes de serem chamadas

Ele deve retornar as posições do vetor onde foi encontrada a substring (trecho de texto ou palavra-chave) fornecida como parâmetro.

Resposta:

function searchKeyword(arr, keywd) {
if (!Array.isArray(arr)) {
console.log("Parâmetro inválido!");
return;
}
const positions = [];
for (let i = 0; i < arr.length; i++) {
if (arr[i].includes(keywd)) {
positions.push(i);
}
}
return positions;
}

Através do async, é possível iniciar uma função e pausar a qualquer momento através do await, que vai buscar o que chamamos de Promises(promessas) que são condicionais para que o código volte a fluir e retorne um resultado, que também pode retornar como uma falha.

return new Promise(resolve =>{

})
}

function func2(number){
return new Promise(resolve =>{
setTimeout(() => resolve(22 + number) , 1000)
})
}

function func3(number){
return new Promise(resolve =>{
setTimeout(() => resolve(14 * number) , 1000)
})
}

async function myAsyncFunction(){
var number = await func1(4);

console.log('Hello');
number = await func2(number);

console.log('World');
var result = await func3(number);

console.log(result);
}
